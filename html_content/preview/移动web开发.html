移动web开发

  • 响应式开发：一套代码运行多个终端 优点:开发快 维护快 适配好 缺点加载速度慢
  • 纯原生移动web开发 一套代码运行一个端 优点: 加载速度快 缺点开发慢 维护慢 适配差

    库和框架的区别

  • 库 便捷的DOM (你想实现什么功能就调用什么方法)(主导者是你 你调用库)
  • 框架：框架 界面工具集 框架是他来控制你 (框架制定好了一系列规则 你按照规则填坑)

        设置视口
        width=device-width 表示设置移动端视口的宽度等于设备的宽度
        initial-scale=1 设置初始化缩放等于1 效果等同于设置宽度
        user-scalable=no 设置用户不允许手动拖放
        minimum-scale=1.0 设置最小缩放等于1
        maximum-scale=1.0 设置最大缩放等于1
    快捷键：meta:vp  + tab

    设置视口的原因：
    1.在PC端的视口：浏览器的可视窗口html的宽度 在PC端是随着浏览器窗口而变化
    2.在移动端的视口 浏览器的html的宽度 固定通常默认是980 但是移动端的设备的宽度只要300-400，980宽度的网页在手机上显示就会出现缩放或者出横向滚动条 会造成网页无法正常显示 需要手指拖放查看
    3.因此，设置移动端的默认视口的宽度html的宽度等于设备的宽度

            *         1. 监听屏幕变化事件onresize
            *         2. 在屏幕变化事件里面获取当前屏幕的宽度
            *         3. 判断屏幕的宽度是否大于640 认为是大屏幕 小于640就是小屏幕

                /*2. 在屏幕变化事件里面获取当前屏幕的宽度*/
                var windowWidth = $(window).width();
                /*3. 判断屏幕的宽度是否大于640 */

                            2. jquery item.data('large-image');//可驼峰也可不驼峰
                        */
                        var pcImgSrc = item.data('large-image');
                        item.html('<a href="#" class="pcImg" style="background-image:url(' + pcImgSrc + ')"></a>');
                    });
                } else {
                    /*小于640就是小屏幕*/
                    var items = $('.carousel-inner .item');
                    items.each(function(index, value) { //index循环索引==i  value循环的值items[i]
                        var item = $(value); //value this 都是dom 转换成jquery对象
                        /*获取自定义属性的值
                            1. 原生JS value.dataset['largeImage'];//要驼峰命名
                            2. jquery item.data('large-image');//可驼峰也可不驼峰
                        */
                        var mobileImgSrc = item.data('small-image');
                        item.html('<a href="#" class="mobileImg"><img src="' + mobileImgSrc + '" alt=""></a>');
                    });
                }
            }).trigger('resize');
        })

## 原生移动web开发方式
- 样式和JS都是原生写的 ，使用流式布局
    - 屏幕尺寸 指的是屏幕对角线的长度
    - 常见的单位： 相对单位：px em pt 绝对单位 in cm
###  网页布局几乎是使用百分比布局(流式布局)
### 像素密度
+ 像素密度是指1英寸里面的像素的数量，通常超过320的像素密度就是高清屏
+ 计算方式：屏幕的分辨率高的平方+屏幕分辨率的宽的平方开根号 除以英寸
### 设备独立像素
+ 表示真实像素和分辨率像素的一个比例  比如3gs手机 高度480px 宽度320px 分辨率  真实设备 高度480px 宽度320px 即分辨率和真实像素一样的时候 1pt=1px
+  但是有一些设备 高度960px 宽度640px 分辨率  真实设备 高度480px 宽度320px 即分辨率和真实像素一样的时候 1pt=2px
### 物理像素和css像素
   - 1. 物理像素(分辨率的像素) 960*640
   - 2.  CSS像素(真实像素)  模拟器里面看见的320*568

+ 清除移动端点击高亮的背景色： -webkit-tap-highlight-color:transparent;

使用滑动事件拖拽元素

    var box = document.querySelector('.box');
    /**
     *     1. 添加滑动开始和滑动中事件
     *     2. 在滑动开始的时候获取滑动开始的X轴坐标和Y轴坐标
     *     3. 在滑动中的时候获取滑动中的X轴的坐标和Y轴的坐标
     *     4. 计算滑动距离 水平 滑动中的X坐标-滑动开始的X轴坐标 垂直 滑动中Y轴坐标-滑动开始的Y轴坐标
     *     5. 把滑动的距离设置到box的位移
     */
    var startX = 0,
        startY = 0,
        moveX = 0,
        moveY = 0,
        distanceX = 0,
        distanceY = 0;
    box.addEventListener('touchstart', function(e) {
        startX = e.touches[0].clientX;
        startY = e.touches[0].clientY;
        console.log(startX + '---' + startY);
    });
    box.addEventListener('touchmove', function(e) {
        moveX = e.touches[0].clientX;
        moveY = e.touches[0].clientY;
        distanceX = moveX - startX;
        distanceY = moveY - startY;
        /*console.log(distanceX+'---'+distanceY);*/
        /*设置位移为滑动的距离*/
        box.style.transform = 'translate(' + (currentX + distanceX) + 'px,' + (currentY + distanceY) + 'px)';
        /*    往右和往下滑动的时候 distanceX 和 distanceY 就是正值
                    往左和往上滑动的时候 distanceX 和 distanceY 就是负值*/
    });
    /*在滑动结束的时候记录滑动的最终距离*/
    /*currentX当前div所在的X的位置
    currentY当前div所在Y轴的位置*/
    var currentX = 0,
        currentY = 0;
    box.addEventListener('touchend', function() {
        currentX += distanceX;
        currentY += distanceY;
    });
